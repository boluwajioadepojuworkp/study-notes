# The 20 Rules of Formulating Knowledge

**By Dr. Piotr Wozniak**: the inventor of spaced repetition and creator of SuperMemo. These rules are the foundation of HOW to structure what you learn for maximum retention. Published February 1999.

> This is THE definitive guide to formulating knowledge for spaced repetition. Every rule below maps directly to our COME STUDIARE method.

---

## The 20 Rules

### 1. Do not learn if you do not understand

Trying to memorize things you don't understand is "blind learning": it takes astronomical time and the knowledge has negligible value.

**Our system:** [[come-studiare/index|Gradient 1 of "I Don't Understand" Protocol]]: if you don't understand, stop. Use the 5-gradient ladder.

### 2. Learn before you memorize

Build an overall picture BEFORE dismembering it into individual facts. Read the chapter first. Then make individual items.

**Our system:** [[come-studiare/index|STEP ZERO: 30-Second Preview]] builds the mental shelf before details.

### 3. Build upon the basics

Simple models first. Basics take a microscopic proportion of your time but each memory lapse on basics costs dearly. You spend 50% of your time repeating just 3-5% of material!

**Our system:** Always start with the simplest explanation. The note template forces "explain like to a 10-year-old."

### 4. Stick to the minimum information principle

The material must be formulated as simply as possible. Simple = easy to remember. Split complex items into sub-items. Example: a paragraph about the Dead Sea → 9 simple Q&A pairs.

**Our system:** [[come-studiare/5-step-method|Step 1: BREAK]]: take only 2 minutes worth. One concept per note. This IS the minimum information principle.

### 5. Cloze deletion is easy and effective

Fill-in-the-blank format: "Bill ...[name] was the second US president to go through impeachment." Quick, effective, recommended for beginners.

**Our system:** This is the natural format for [[templates/note-template|our note template]]: "What it is: (one sentence)." The gaps ARE the retrieval practice.

### 6. Use imagery

A picture is worth a thousand words. Visual cortex is vastly superior to verbal processing. Use maps, diagrams, mind maps.

**Our system:** [[come-studiare/index|Gradient 5 of "I Don't Understand"]]: DRAW it. Rectangle, arrow, stickman. The Quartz graph IS the ultimate imagery.

### 7. Use mnemonic techniques

Mind maps, peg lists, memory palaces. With training, these become automatic for 1-5% of items. The bottleneck is NOT in quickly memorizing: it's in retaining for months/years.

**Our system:** [[come-studiare/index|Episodic memory techniques]]: physical anchors, emotional hooks, location rotation. All are mnemonic techniques.

### 8. Graphic deletion is as good as cloze deletion

Cover parts of an image and ask: "What is this?" One illustration can generate 10-20 items. Great for anatomy, geography.

**Our system:** Use the Quartz graph: each node you DON'T link to yet IS a "missing part" of your knowledge picture.

### 9. Avoid sets

A set is a collection without order (e.g., "What countries belong to the EU?"). Sets are virtually impossible to memorize. Convert to enumerations or ordered lists.

**Our system:** Never list concepts without structure. Each concept gets its OWN note. Links create the order.

### 10. Avoid enumerations

Even ordered lists are hard. Use overlapping cloze deletions to break them into pieces. The alphabet example: "What three letters does the alphabet begin with?"

**Our system:** [[come-studiare/3-pass-system|Pass 2: Cluster]]: recall 5 sentences in order. This IS overlapping cloze for enumerations.

### 11. Combat interference

Similar items CONFUSE each other. Memory interference is the #1 cause of forgetting. Make items unambiguous. Eliminate interference as soon as you spot it.

**Our system:** [[come-studiare/index|Implicit Memory: Interleaving]]: mix problem types. Compare side-by-side. "Spot the difference" tables prevent interference.

### 12. Optimize wording

Reduce complex sentences to smart, compact maxims. Fewer words = faster learning. The exact wording matters for retrieval speed.

**Our system:** [[come-studiare/5-step-method|Step 4: Write ONE sentence]]. Not a paragraph. ONE sentence. Maximum compression.

### 13. Refer to other memories

Link new knowledge to existing memories. "Cringing = shamelessly humble and supplicant": uses words you already know. Builds a coherent, hermetic knowledge structure.

**Our system:** [[wikilinks]] between notes ARE references to other memories. The entire knowledge graph is one big "refer to other memories" system.

### 14. Personalize and provide examples

Personal references are extremely resistant to interference. "Like the divan at Robert's parents' house" is FAR more memorable than a dictionary definition.

**Our system:** The note template REQUIRES "My example: (a concrete, stupid example)." This is personalization built into every note.

### 15. Rely on emotional states

Emotions are linked to memories. Vivid, shocking, funny examples stick. Use love, war, your late relative, a celebrity: anything that evokes emotion.

**Our system:** [[come-studiare/index|Dopamine System]]: chocolate, music, dance. Positive emotions during encoding = stronger memories. The Dopamine Journal builds emotional connection.

### 16. Context cues simplify wording

Use labels, prefixes, templates to set context instantly. "bioch: GRE" instead of "What does GRE stand for in biochemistry?": saves milliseconds, prevents wrong brain areas from activating.

**Our system:** Every note has frontmatter (title, course, date). The course acts as the context cue. `[[wikilinks]]` ARE context cues.

### 17. Redundancy does not contradict minimum information

Passive + active recall, reasoning cues, multiple representations, derivation steps: all valid forms of redundancy that STRENGTHEN memory without violating simplicity.

**Our system:** [[come-studiare/5-step-method|Step 3 + Step 4]]: you recall aloud AND write. Two representations. Also: voice + writing = dual coding = productive redundancy.

### 18. Provide sources

Except for proven facts (2+2=4), always include where knowledge came from. Sources help you update knowledge when it becomes obsolete. Include reliability labels.

**Our system:** Every note has a "Source" in the template. Every reference note links to the original URL. Date stamping built in via frontmatter.

### 19. Provide date stamping

Knowledge can be stable (math) or volatile (tech specs). Date stamp items. When knowledge changes, you know what to update.

**Our system:** Every note has `date` in frontmatter. Quartz shows "last modified." The recall schedule IS date stamping for review.

### 20. Prioritize

You will always face more knowledge than you can master. Prioritize: sources → extracting → transferring → formulating → using forgetting index → learning tools.

**Our system:** [[come-studiare/index|Terrible Brain Day Protocol (Green/Yellow/Red)]] IS prioritization. The recall schedule IS a priority system. Emergency cramming IS last-resort prioritization.

- Simple words ("like to a 10-year-old")
- One sentence per concept
- Draw it if words fail
- Link to what you know
