# The Complete Note-Writing System — Andy Matuschak

**Source:** [§Note-writing systems](https://notes.andymatuschak.org/zEZi3ytBhZHHweRYNAosQ1g)

This is the master outline of Andy Matuschak's entire note-writing philosophy. Every principle, every workflow, every structure decision. Use this as a checklist for your own system.

---

## 1. Core Principles

| Principle | What It Means |
|-----------|--------------|
| **Evergreen note-writing as fundamental unit of knowledge work** | Writing notes IS the work. Track "notes written per day" as your metric. |
| **"Better note-taking" misses the point** | The goal is better THINKING, not better notes. |
| **Knowledge work should accrete** | Ideas should accumulate over time, not get lost. |
| **Note-writing practices are generally ineffective** | Most people's notes are transient — they don't add up to anything. |
| **Note-writing practices provide weak feedback** | Evergreen notes create tighter feedback loops because you constantly revise past notes. |

## 2. Reading → Note-Writing Pipeline

| Step | Action |
|------|--------|
| **Write about what you read** | To internalize texts deeply — don't just highlight. |
| **Reading efforts accumulate** | Externalized notes remove the limit of your memory. |
| **Collecting material feels more useful than it is** | Beware: collecting is NOT learning. |
| **Do your own thinking** | Don't just summarize the author. Add YOUR insight. |
| **Understanding requires effortful engagement** | If it feels easy, you're not learning. |
| **Writing forces sharper understanding** | You don't really know it until you can write it. |
| **Literature notes are secondary and separate** | Keep reading notes separate from your thinking notes. |

## 3. Capture System

| Tool | Purpose |
|------|---------|
| **Writing inbox** | For transient and incomplete notes — capture without friction. |
| **Reading inbox** | For possibly-useful references. |
| **One inbox, not many** | Inboxes shouldn't be split into multiple places. |
| **No automatic import** | Beware automatic import into the reading inbox. |
| **Phone = collect and triage** | Use phones to capture, not (usually) to read. |
| **Pocket memo pad** | Physical capture when out — feeds into writing inbox. |

## 4. Thinking → Note-Writing

| Principle | Action |
|-----------|--------|
| **Insight accumulates** | Evergreen notes build on each other over time. |
| **Spaced repetition develops inklings** | Use SRS to incubate half-formed ideas. |
| **Leaps of insight from prior thought** | Big ideas come from connecting small notes. |
| **Brainstorming = missing accretion system** | If you need brainstorming, you haven't been building notes. |
| **Safe place for wild ideas** | Evergreen notes nurture wild ideas incrementally. |
| **Conversational bandwidth** | Share a note instead of explaining from scratch. |

## 5. Writing Output

| Technique | How |
|-----------|-----|
| **Executable strategy for writing** | Outline = shuffle note titles. Draft = edit notes together. |
| **Speculative outlines while writing** | Create outlines early, fill in notes over time. |
| **Lower emotional stakes** | Deleting a note ≠ deleting a manuscript chapter. |
| **Incremental writing** | Small, self-contained notes = regular checkpoints. |

## 6. Note Structure Rules

| Rule | Why |
|------|-----|
| **Atomic** | One note = one concept. Not a page. Not a summary. |
| **Concept-oriented** | By idea, not by book/course/author. |
| **Densely linked** | Links > folders. The web IS the organization. |
| **Titles are APIs** | Complete declarative sentences. You can reference the idea by name. |
| **Prefer positive titles** | "X enables Y" not "X doesn't work." |
| **Notes should surprise you** | If every note confirms what you know, you're not learning. |

## 7. Associative Structure (No Folders)

| Principle | Why |
|-----------|-----|
| **Associative > hierarchical** | The brain works by association, not by folders. |
| **Tags are ineffective** | Tags become inconsistent and unmaintained. Links are better. |
| **Indexed references > tags** | A note listing related notes is more useful than a tag. |

## 8. Tools Philosophy

| Principle | Why |
|-----------|-----|
| **Plaintext** | Timeless, portable, future-proof. |
| **Plaintext IDs to link** | Stable identifiers for linking between notes. |
| **Human as connector** | The system has no app. YOU are the intelligence connecting ideas. |
| **Digital tools ignore inter-note sense-making** | Most apps focus on individual notes, missing the big picture. |

## 9. History

| Source | Contribution |
|--------|-------------|
| **Zettelkasten (Luhmann, 1960s)** | Original slip-box: atomic notes, cross-referenced by hand. |
| **How to Take Smart Notes (Ahrens, 2017)** | Modern revival: Zettelkasten for academic writing. |
| **Evergreen Notes (Matuschak, 2020+)** | Evolution: concept-oriented, constantly revised, public by default. |

## How This Maps To Our System

| Matuschak Principle | Our Implementation |
|---------------------|-------------------|
| Atomic notes | Each `.md` file = one concept |
| Concept-oriented | Notes in `content/` organized by idea, cross-linked |
| Densely linked | `[[wikilinks]]` throughout all notes |
| Writing inbox | Keep a `_inbox.md` or use `tools/highlight-to-note.sh` |
| Spaced repetition | Our [[come-studiare/recall-schedule|recall schedule]] |
| Plaintext | Everything is Markdown |
| Executable writing strategy | Notes → Quartz builds → published site |
| Associative structure | Quartz graph view + backlinks (no folders, only links) |

> **Key difference:** Matuschak develops NEW ideas. We learn EXISTING curriculum. Same principles, different goal. Our system can evolve into his over years.
