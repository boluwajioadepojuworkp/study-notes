# Michael Neuper's Note-Taking Method

**Source:** [How I Use Org-Roam for CS](https://michaelneuper.com/posts/how-i-use-org-roam-to-takes-notes-for-cs/) — 22 November 2024

**What it is:** A Zettelkasten-style note system using Org-Roam in Emacs for a CS degree. Notes are written FROM MEMORY after lectures, not during them.

**How it works:**
1. Attend lecture — LISTEN, don't write notes. Focus on understanding.
2. Same evening — write notes FROM MEMORY. What do you remember?
3. Fill gaps using textbook and lecture slides.
4. Structure: Index → Year → Semester → Class → Week/Chapter → Topics.
5. Topics link to other topics via `[[wikilinks]]`. The graph grows naturally.
6. Code and LaTeX are embedded directly in notes (executable code blocks).

**Why it connects:**
- The "write from memory" step is [[come-studiare/5-step-method|Step 3 of the 5-Step Method]] — retrieval practice.
- The topic-linking is elaboration — building [[come-studiare/index|semantic memory]].
- The same-day review matches the [[come-studiare/recall-schedule|recall schedule]] (before sleep).
- This is the student-optimized version of [[notes/matuschak-evergreen-notes|Andy Matuschak's Evergreen Notes]] and the original Zettelkasten by Niklas Luhmann.

**My adaptation:** We use Markdown + Quartz instead of Org + Emacs. Same principles, simpler tools.
